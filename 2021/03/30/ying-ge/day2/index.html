成员变量和局部变量区别

类中位置不同

  • 成员变量 类中方法外，即只存在于类中，不存在与任何方法中

  • 局部变量 方法内或者方法声明上

    public void eat(String a)  //String a即为方法声明

在内存中的位置不同

  • 成员变量 堆内存 （即new出来的空间）
  • 局部变量 栈内存

生命周期不同

  • 成员变量 随着对象存在而存在，随着对象消失而消失
  • 局部变量 随着方法的调用而存在，随着方法的调用完毕而消失

初始化值不同

  • 成员变量 有默认初始化值 （可以通过 类.属性的形式获取变量）

  • 局部变量 无默认初始化值，必须先定义，赋值，才能使用

  • 多个类中存在相同属性和行为，将这些内容抽取到单独一个类中，多个类只需要继承这个类即可

  • 子类可以继承父类所有属性和方法

  • 一个子类只能继承一个父类，一个父类可以被多个子类继承

    (一个儿子只有一个爸爸，一个爸爸有多个儿子)

  • 在代码层面，如果一个类没有继承任何父类，那么这个类将默认继承Object类

重写和重载的区别

  • 重载：在同一个类中，允许方法名相同，但参数列表不同
  • 重写：当子类继承父类后，父类功能不满足子类需求，子类可以修改父类的逻辑

  • super

转型

  • 向上转型：父类的引用指向子类的对象

    • 调用子类中重写父类的方法
    Animal animal = new Rabbit();//Animal的引用具体指向Rabbit()这个实例

  • 向下转型:父类的引用转为子类对象 A a –> B a

    • 用处：子类中除了拥有父类的方法，还拥有自己的方法

    ，因此是由功能较强的类型指向功能较弱的类型。

    ​ 父类的引用这个对象实例在内存的本质上还是子类类型的

    只是new 子类的引用类型被设置为了父类。

    ​ 

    Son son = (son)father  //father原来是Father类型，此处将father转换为son类型
    变量son引用了变量father的地址，而father引用的对象地址是new Son()创建的Son对象，这里只是将引用的类型强制转换成了Son,本质引用地址所存储的对象就是Son类型的

    ​

    • 向下转型抛出异常

      • 父类强制转换为子类并不是总成功的，当引用类型的真实身份是父类本身类型时，强制类型转换就产生错误。

        Father father = new Father();
Son son = (Son)father

      • 异常原因：

        • 子类强制转换成父类时：此时Son类型对象的引用是Father类型，由于Son类型是对Father类型的扩展，调用Ftaher引用的方法在Son对象中是存在的，可行
        • 父类强制转换成子类时：父类引用的对象时Father类型，强制转换之后,Father类型对象的引用是Son类型的，被引用的对象仍然是Father类型的，Son扩展的对象在Father中并不存在，无法调用，产生报错。

接口

  • 接口不给出具体实现，只包含抽象类
  • 接口等同于抽象类的简化
  • 例如 计算机认识的是USB标准，但是不关心这个标准的具体实现子类。
  • 接口中只有常量和抽象方法

接口特点

  • 用关键字 interface表示

    • 格式：interface 接口名 {}

      public interface Person {
}

  • 类实现接口用 **implements**表示 (接口与类的实现方式，即继承关系)

    • 格式：class 类名 implements 接口名{}
    • 接口 实现，类 继承
  • 接口不能实例化
    • 接口按照多态的方式，由具体的子类去完成实例化，所以也可以说是接口多态
  • 接口的子类
    • 抽象类
    • 重写接口的所有抽象方法
  • 接口可以与多态一样，父类的引用指向子类的对象

接口成员特点

  • 成员变量
    • 只能是常量
      • 默认修饰符 public static final
      • 默认修饰符可省略
  • 构造方法
    • 接口没有构造方法，接口主要是扩展功能，没有具体存在，
  • 成员方法
    • 只能是抽象方法
    • 默认修饰符 public abstract，一般情况下可省去

  • 继承关系，可以单继承，也可以多继承(使用,隔开)

    • 因为 实现 关系必须要重写方法，而两个接口上实际上是两个只有抽象方法的抽象类，无法重写方法
    public interface MyInterface extends MyInterface2,MyInterface3

抽象类和接口区别

  • 成员区别
    • 抽象类 ： 变量，常量；有抽象方法，非抽象方法
    • 接口 ： 常量，抽象方法
  • 关系区别
    • 类与类 继承：单继承
    • 类与接口 实现，单实现，多实现
    • 接口与接口 继承，单继承，多继承
      • public interface Person extends Animal,give{}
      • 接口可以继承多个接口，用,隔开即可
  • 设计理念区别
    • 抽象类 被子类继承体现的是 ： is a的关系，共性功能
      • 例 猫是一个动物
    • 接口 被子类实现 体现的是： like a的关系，扩展功能
      • 例 豆腐像甜品

final关键字

  • final :表示最终的，可以修饰类，属性和方法

  • 如果用final修饰属性，必须给这个属性初始化值

  • 基本数据类型 在程序允许的时候值是不可修改的

  • 引用数据类型，在程序允许是，不可指向别的内存地址

  • 如果用final修饰类，这个类不能被子类继承

  • 如果用final修饰方法，这个方法不能在子类中被重写

常量

  • java常量的声明

    • static + final声明

      public static final String COUNTRY_NAME = "中国"  //常量一般用pubic修饰

    • 常量不需要多次加载

      • 如果是成员变量，则是实例（对象）级别的，每次都需要加载。
      • static是类级别的，随着字节码文件加载而加载，在类加载时之加载一次，占用空间最小

常量命名

  • 常量名称所有字母都要大写!!
  • 有多个单词时用__隔开

包

  • 实际上是文件夹
  • 作用： 对类进行分类管理

package

​ 当前类所在包路径

package com.wufan.interface_test.personTest;

import

  • import -> 导包 引入第三方的类

    import java.util.Scanner
  • java.lang包下的类不需要再次导入，在底层已经导入

四种访问权限修饰符

  • 访问权限越小，安全性越高
  • public ：公共访问权限修饰符，在当前工程可以访问
  • private：私有访问权限修饰符，只有在本类才能访问
  • default ：默认受保护的，只能在当前包才能访问，子类以及子包不可访问
  • protect 受保护的，只能在同包中进行访问，要想在其他包中使用，必须创建其子类。

内部类

内部类定义

  • 将类写在其他类的内部，可以写在其他类的成员位置和局部位置，

    这种类称为内部类

内部类可以使用外部类的属性和方法，如果存在属性或者方法同名，则使用this指向外部类

定义方法

外部类.内部类 引用 = new 外部类对象.new 内部类对象

内部类使用

  • 当一个事物里包含着其他事物的时候，其他事物可以创建一个内部类。

内部类位置

  • 成员位置（成员内部类） 即在类中定义的类
  • 局部位置（局部内部类） 即在方法中定义的类

匿名内部类

匿名内部类是内部类的简化写法

  • 前提：存在一个类或者接口
    • 这里的类可以是具体类也可以是抽象类
  • 格式：

    • new 类名或接口名(){重写方法}

      //调用操作账户的方法  创建接口的匿名对象
        account.updateAcco  unt(52.5,new AccountTest(){


            @Override
            public void saveAccount(double money) {
                System.out.println("保存了"+money);
            }
        });
  • 本质
    • 是一个继承了类或者实现了接口的子类匿名对象
    • 匿名内部类是子类的匿名对象
