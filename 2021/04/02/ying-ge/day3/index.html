day3_成员变量+内存分布

发表于 2021-04-02 更新于 2021-08-05 分类于 后端 ， java

摘要

成员变量、局部变量、静态代码块、方法的内存分布

static

​ static 表示静态(类级别的)，可以修饰方法，变量，如果修饰变量，我们称之为”静态变量“，静态变量随着类加载而加载，并且只加载一次

  • 普通的局部变量 存放在栈内存中
  • 静态变量 随着类加载时加载（编译class文件时），存放在方法区中

类级别的变量存放在方法区中

实例（对象）

成员变量 ，局部变量

  • 成员变量 定义在类中
    • 普通成员变量 ：创建实例类型(new关键字)时加载，无论是引用数据类型还是基本数据类型都存放在堆内存中。

  • 局部变量

    • 定义在普通方法中和静态方法中。

      • 引用数据类型 左边声明部分存放在栈内存中，右边赋值部分存放在堆内存中。
      • 基本数据类型 变量的声明和赋值都存放在栈内存中。
      String name = "wa"  //name存放在栈内存中，wa存放在堆内存中。

静态代码块

  • 类级别的区间

  • 静态代码块也是随着类的加载而加载，以后常用于类的初始化

  • 静态代码块会在主方法执行之前执行

    public class StaticTest02 {

            public static void main(String[] args) {

                System.out.println("main执行了");
            }

    //静态代码块
    static {

        System.out.println("静态代码块执行");
    }
}

//结果
静态代码块执行
main执行了

方法的内存分布

  • 无论是普通方法还是静态方法，方法的分布只能是栈中
