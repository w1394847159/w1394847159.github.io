day4_异常

发表于 2021-04-05 更新于 2021-08-05 分类于 后端 ， java

摘要

本文介绍了java常用的各种异常。

异常

  • Java程序运行过程中发生的异常事件：
    • 错误（Error）:JVM系统内部错误，资源耗尽等严重情况
      • 较严重，在代码层无法处理
        • 虚拟机错误 VirtualMachineErroe
        • 内存溢出 OuyOfMemoryError
        • 线程死锁 ThreadDeath
    • 异常(Exception):其他因编程错误或偶然的外在因素导致的一般性问题
      • 在代码层可处理
        • 非检查异常 编译器不要求强制处理的异常 RuntimeException
        • 检查异常 java要求必须要检查的异常

常见异常

  • RuntimeException
    • ArithmeticException :数学计算异常
    • NullPointerException空指针异常
    • ArrayIndexOuyOfBoundsException数组下标越界异常
    • ClassCastException类型转换异常

异常分类

  •   编译时异常：基本上发生错误概率较高，java将其划分为编译时异常，要去开发人员一定要进行处理

  •   运行时异常：发生概率较低，可处理可不处理
  • 在代码层面判定编译时异常和运行时异常，就要看当前异常是否继承RuntimeException，如果继承了就是运行时异常，如果没有继承即为编译时异常。

异常处理机制

  • JAVA中异常处理通过5个关键字实现： try,catch,finally,throw,throws
    • 捕获异常
      • try 执行可能产生异常的代码
      • catch捕获异常
      • finally无论是否发生异常代码总能执行
    • 声明异常
      • throws声明可能要抛出的异常
    • 抛出异常
      • throw手动抛出异常

try - catch方法

  • try{放入需检测的代码块}catch(exception的引用){}final{无论如何都会执行的代码块}

    • 在当前位置处理异常。

      public void method(){
    try{
        //代码段1
        //产生异常的代码段2
    }catch(异常类型 ex){
        //对异常进行处理的代码段3
        System.out.println(e.getMessage());  //getMessage()方法是输出异常描述信息。
    }finally{
        //无论如何都能执行的代码段
    }
}

//在try中放入需要检测的代码块，如果出现了异常
//则交由catch处理，如果不对异常处理，会造成程序中断，
//当catch处写入多个异常时，子类的异常不能写在父类下面

  • try块后可接0个或多个catch块，如果没有catch块，必须跟一个finally块。

throws

  • 语法格式 : 在方法名之后: throws 异常类型

  • 如果一个方法会出现异常或无能力处理异常，可以在方法声明处用throws子句来声明抛出异常

    • 当方法抛出异常列表里的异常时，方法将不对这些异常进行处理，而抛向调用该方法的异常

      谁调用，谁处理

  • throws语句在方法定义时声明该方法要抛出的异常类型，可以有多个异常，如果出现没有声明的异常类型，java虚拟机会自动执行下去，出现异常则报错，程序终止。

    public FileInputStream getFile() throws FileNotFoundException {}
    • 如果throws Exception，那么在调用方法处处理异常时需要声明异常类型为 Exception，因为方法处使用了Exception,检测到异常后，如果没有对应处理方法，则异常依旧会执行。
      • 即声明什么异常，处理什么异常

throw

  • throw用来抛出一个异常
    • 例如： throw new IOException();
  • throw抛出的只能够是可抛出类Throwable或者其子类的实例对象
    • 即：自定义异常时构造的异常方法实际上就是Throwable的子类

  • throw使用方法

    法1
public void method(){
    
    try{
        //代码段
        throw new 异常类型();
        
    }catch(异常类型 ex){
        //对异常进行处理的代码段2
    }
}

法2
    public void method() throws 异常类型{
    //代码段1
    throw new 异常类型();
}
  • throw抛出异常对象的处理方法：
  • 1.自己异常，自己处理，在原处使用try-catch方法
  • 2.向上抛出，在方法声明处添加throws Exception,在方法调用处异常处理
  • 此时可以抛出与throw对象相同的类型或者父类，不能是子类

自定义异常

原因

  • 以后在做开发的时候，java给我们提供的异常是不够的，要根据业务，自定义异常。

自定义异常步骤

  • 自定义编译时异常：类使用 extends 继承Exception

    • 创建有参构造，指向父类

      //创建一个编译时异常，Exception这个父类实际上继承了Throwable,message传递给了Throwable 
//定义一个异常,负数时 运行异常
public class Student_Exception extends RuntimeException{

    //创建一个新的异常构造方法
  //由于类
    public Student_Exception(String message){
        super(message);
    }
      • 需要使用异常的方法声明异常类型
        • 抛出异常 throw 自定义的异常

  • 自定义运行时异常：类继承RuntimeException

    //创建一个编译时异常，Exception这个父类实际上继承了Throwable,message传递给了Throwable
   public ExceptionTest05(String message){
       super(message);
   }
    • 同上

      super(message);
