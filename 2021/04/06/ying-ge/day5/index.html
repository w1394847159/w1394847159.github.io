摘要

自动装箱、自动拆箱、集合、哈希表

自动装箱

  • 自动将基本数据类型转换为引用数据类型
  • 在集合中通常都会将数据自动装箱为引用数据类型

  • java中所有的基本数据类型（int,double,float,double,String,char,short,long）都有对应的包装类

  • 除了int –>Interger ,char–>Character外,其余所有数据类型自动装箱皆为首字母大写

  • int 基本数据类型 –> Integer 包装类

    //自动装箱
Integer total = 99;
等同于
Integer total = Integer.valusOf(99);

//自动拆箱
int total1 = total;
int total1 = total.intValue();

  • double基本数据类型 –>Double包装类

    //double 基本数据类型 可浮点型可不浮点型
double a = 5;

//Double 引用数据类型 必须写成浮点型
Double a1 = 5.0;

  • char 基本数据类型 –>Character包装类

    //char基本数据类型 
char a = 'a';

//char 引用数据类型 写法与char

自动拆箱

  • 自动将包装器类型转换为基本数据类型

  • 将引用数据赋值给基本数据类型时，会将引用数据类型拆解为基本数据类型

    Double d2 = 5.0;
double d3 = d2; //此时实际上是将值传递给d3,而不是地址

  • int —-> Integer

    //将656转换成二进制字符串
String i = Integer.toBinaryString(656);

集合

  • 集合是一个容器，可以存放所有类型的数据，而且是不同数据类型

    集合只能存放引用数据类型，但基本数据类型可以使用自动装箱存入集合

Collection

  • 集合的顶层父接口，没有直接实现类
    • Set:Collection的子接口，不记录元素位置，无序状态,不可重复，查找效率低
    • List：Collection的子接口，是一个有序的集合，可包含重复的元素，提供了按索引访问的方式。

List

List接口下有三个实现类 ：ArrayList,LinkedList,Vector

  • Vector 线程同步的动态数组，效率很低
  • ArrayList 实现一个动态数组，它的规模可变并且可以和链表一样被访问，允许快速随机存取，读快改慢
  • LinkedList 提供最佳顺序存取，适合插入和移除元素。改快读慢

ArrayList

  • ArrayList是List接口的一个子类，也是一个具体实现类

    //定义一个类对象
ArrayList arrayList = new ArrayList();

ArrayList,Vector异同

相同点

底层封装了数组，具备数组的所有特性，数组可以通过查找下标读取元素，因此数组查询效率较高，但增删改效率较低。

不同点

ArrayList:

  • 线程不安全的list集合，效率比较高（以后开发中可以通过别的途径解决线程问题）
  • 扩容机制： ArrayList初始长度为10，扩若时按1.5倍扩容 10* 1.5 = 15 (创建对象的时候可以改变其长度 )

Vector:

  • 线程安全的list集合，效率较低（因为在其方法中都添加了线程synchronized关键字）

  • 扩容机制：Vector初始长度为10，扩容时按2倍扩容 10*2 = 20

LinkList

  • LinkList底层封装了链表
  • 以节点的形式存放数据，随机增加，删除，修改，效率较高，查询效率较低！改快读慢

List常用方法

  • list.add()

  • 作用：添加数据 可添加任意数据，只要是引用数据类型即可，包括添加类对象

  • 只有List才有此方法，set是无序集合，无此方法，所以父类Collection也无此方法

//实例化对象 由于Collection是接口，无法实例化对象，需要指向其子类（的子类）

//添加数据 实例.add();
collerction.add(10);  //自动装箱，引用数据类型 Integer i = Integer.valusOf();
  • list.clear 清空集合 此方法是所有集合都拥有的方法
  • 作用：清空集合
  • boolean empty = list.isEmpty()
    • 作用：判断当前集合是否为空
    • 由于是布尔类型的返回值，需要用布尔类型接收
  • list.remove(集合中元素)
    • 作用：删除集合中的元素
    • 注意：在遍历集合时不能更改集合的元素，否则会改变迭代器状态，产生报错

遍历集合方法

  • 迭代器循环
    • 在迭代器中无法使用集合自身方法更改集合，包括增删改，必须要使用迭代器自身方法进行更改，更改之后，重新遍历才会产生结果
    • 迭代器在使用之前，必须获取迭代器对象
    • 迭代器方法：
      • iterator.hasNext() 如果仍有元素可以迭代，返回true
      • iterator.next() 返回迭代的下一元素，指针继续移动一格
      • iterator.remove()迭代器自身的删除方法，删除后需要重新迭代才可看出变化
  • 加强for循环
    • for(ElementType element:arrayName){}
    • ElementType 元素类型 element 元素变量
    • arrayName 需要循环的集合
法1
//获取迭代器对象
Iterator iterator = collection.iterator();

//判断集合内下一个元素是否为空，最开始没有指向任何元素，如果不为空，返回true
while(iteratoer.hasNext()){
    
    //.next返回下一节点的元素，指针往下走一格
    Object next = iterator.next();
    
    System.out.println(next);
    
}

Set

  • Set接口有HashSet和TreeSet两个实现类
  • HashSet可以快速定位一个元素，无序不可重复，底层是HashMap,存放的值在HashMap的Key部分
  • TreeSet 将放入其中的元素按序存放，加入了对象比较的方法。通过对集中的对象迭代，可以得到一个升序的集合。

HashSet

  • 作用： 快速定位一个元素
  • 特点：无序不可重复，底层是HashMap,存放的值在HashMap的Key部分

TreeSet（未完，与HashSet一样写，并加入比较方法）

  • 作用：将放入其中的元素按序存放。
  • 特点：拥有对象比较方法，通过对集中的对象迭代，可以得到一个升序的对象集合。
    • 自定义类的话必须要实现比较接口，重写比较规则，底层是TreeMap，存放的值是TreeMap的key部分。

自定义类的比较方法

  • 自定义类必须要实现Comparable接口
  • 自定义类必须要重写compareTo方法
    • 如果是整型数据，通过二叉树判断，返回结果大于0，比较值放在被比较值后(正序)；返回结果小于0，比较值放在被比较值前（逆序）。

泛型

  • 泛型： 强制集合只存放某一种数据类型，即不再需要强制转换
  • 作用：在集合迭代时不需要向下转型

用法

 //采用泛型创建集合对象  
//<>称为钻石表达式，写入值存放的数据类型。
 ArrayList<Animal> animals = new ArrayList<>();

泛型好处

  • 代码更加简洁【不用强制转换】
  • 程序更加健壮【只要编译时期没有警告，那么运行时期就不会出现ClassCastException异常】
  • 可读性和稳定性【在编写集合的时候，就限定了类型】

Map接口

  • Map接口有HashMap,HashTable和TreeMap等实现类

Map实现类选择

  • 元素顺序很重要时选 TreeMap，
  • 元素不比以特定顺序存储时使用HashMap。
  • HashMap不同步，HashTable同步，HashMap提供了所有HashTable类似功能，速度更快。

hashMap

  • HashMap的key是不允许重复的，value可以重复，value只是Key的替代品
  • hashMap的值可以为空

hashMap存储过程

调用put方法后，底层调用hashCode生成hash值。

hash值会确定当前节点存放在数组的哪个位置上，如果当前数组节点没有元素的话，就直接存入；

如果有元素，则根据当前节点node的key依次进行比较(equals).如果key相同，新的节点覆盖旧的节点，这也是hashMap方法不允许重复的原因。

TreeSet

  • TreeSet是Set方法的具体实现类

  • 通过TreeSet存放元素会自动排序

  • TreeSet会根据字符串首字母进行排序，不区分大小写

Properties

  • Properties也是Map集合，但是只能是String类型。

哈希表

  • 哈希散列表：根据关键码值(Key value)而直接进行访问的数据结构，通过把关键码值映射到表中的一个位置来访问记录，以加快查找的速度

  • hash散列表是由数组和单向列表组成：

    • 数组优点：查询效率高，随机增删效率低
    • 链表的优点：随机增删效率高，查询效率低

  • 哈希散列表 拥有数组和链表的优点，查询效率高，随机增删

工作原理

  • 通过hashcode将键转化为相应的哈希值，要查找键时，通过哈希值查找
