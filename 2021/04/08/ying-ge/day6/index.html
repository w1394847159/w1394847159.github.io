Map接口

引用

  • Map接口和Collection接口是平级的，包含了Key-value对。

Map方法

  • map.put(键，值) 添加元素

    • 在此映射中关联键与值
    map.put("name","jack") "name"是键，"jack"是值

  • map2.putall(map)

    • 将一个集合的元素传给另一个集合，即复制集合中的所有内容
  • int size = map.size()
    • 获取集合的长度

  • containsKey(Object key)

    • 判断集合是否包含某一个Key，包含返回true
    containsKey("name")
  • containsValue(Object value)
    • 判断集合是否包含某一个value

  • map.equals(map2)

    • 判断两个集合的内容是否相同，相同返回true
    map.euqals(map2)

  • map.get(Obeject Key)

    • 根据key的值获取value
    Object o = map.get("name");

  • map.remove(Obeject Key)

    • 根据Key删除value,并返回value
    Object o = map.remove("name")

      Set<String> strings = hashMap.keySet();  //keySet方法 返回所包含的键的集合，返回值类型为Set

    2. 创建迭代器

      Iterator iterator1 = strings.iterator();

    3. 进行遍历

      while (iterator1.hasNext()){
    //通过next方法获取键
    Object name = iterator1.next();
    
    //通过get方法获取值
    Object values = hashMap.get(name);
    
    //输出对应关系
    System.out.println(name +"="+ values);

hashMap

  • HashMap的key是不允许重复的，value可以重复，value只是Key的替代品
  • hashMap的值可以为空

hashMap存储过程

调用put方法后，底层调用hashCode生成hash值。

hash值会确定当前节点存放在数组的哪个位置上，如果当前数组节点没有元素的话，就直接存入；

如果有元素，则根据当前节点node的key依次进行比较(equals).如果key相同，新的节点覆盖旧的节点，这也是hashMap方法不允许重复的原因。

TreeSet

  • TreeSet是Set方法的具体实现类

  • 通过TreeSet存放元素会自动排序

  • TreeSet会根据字符串首字母进行排序，不区分大小写

Properties

  • Properties也是Map集合，但是只能是String类型。

哈希表

  • 哈希散列表：根据关键码值(Key value)而直接进行访问的数据结构，通过把关键码值映射到表中的一个位置来访问记录，以加快查找的速度

  • hash散列表是由数组和单向列表组成：

    • 数组优点：查询效率高，随机增删效率低
    • 链表的优点：随机增删效率高，查询效率低

  • 哈希散列表 拥有数组和链表的优点，查询效率高，随机增删

工作原理

  • 通过hashcode将键转化为相应的哈希值，要查找键时，通过哈希值查找
