day7_Tomcat+servlet

置顶 | 发表于 2021-04-09 更新于 2021-08-05 分类于 后端 ， java

摘要

本文介绍Servlet + Tomcat服务器

Servlet

  • Servlet是JavaEE规范之一
  • 是运行在服务器上的一个java小程序，可以接收客户端发送过来的请求，并相应数据给客户端

Servlet的运行流程：

​

  • Servlet一般是由浏览器发起调用的，在浏览器发起调用的时候会创建servlet的实例，servlet实例由容器创建的(tomcat服务器)

    调用构造方法即创建当前类的实例

  • 1.先由浏览器发起调用，tomcat服务器在接收到浏览器的请求之后，会创建servlet对应资源的实例，

  • 2.创建完servlet实例，会调用默认的初始化方法init()方法

  • 3.初始化方法执行完毕，一定会执行servce方法 ， 请求分发处理方法，根据浏览器不同的提交类型执行不同的方法

    通过request接收前端表示
    String flag = req.getParameter("flag");

//根据前端接收到的表示，完成请求的分发处理
if (flag != null && flag = "list"){
    
    //执行对应方法
}else if (flag != null && flag = "addlist"){
    //执行另一方法
}

  • 4.当tomcat服务器关闭的时候，执行销毁的方法destroy()

Tomcat

  • tomcat 是一款服务器，java包需要上传tomcat才可使用
    • 要想使用必须要有环境变量

使用Tomcat

  1. 创建一个新文件夹，存放Tomcat安装包
  2. 在环境变量中新建JAVA_HOME，编辑值为jdk安装目录，一般在C://Program Files/JAVA中
  3. 在Tomcat bin目录下点击startUp.bat启动，shutDown.bat停止

Tomcat文件包

  • bin 专门用来存放Tomcat可执行程序

  • conf 专门用来存放配置文件

  • lib 专门用来存放jar包

  • webapps 专门用来存放部署的web工程

  • work Tomcat 工作室目录

  • conf/server.xml 可更改端口号

Tomcat启动方式

  • 在bin目录下点击startup.bat
  • 在bin目录下使用catalina run命令

jar包

每个Jar包是对应的组织封装的工具类

  • servlet包完成前后端交互
  • taglibs包添加样式

搭建Web工程步骤

创建Web工程

  • new 一个普通java工程

  • 对创建的工程右击，选择Add Framework Suppot,导入动态web依赖

  • 选中 Web appliction 动态web，会出现以下文件

    • WEB-INF是受保护的文件，外界不可访问，一般用于导入第三方类
    • web文件夹主要是写一些静态的前端页面

导入jar包

  • 把存放jar包的目录放在web-info下

  • 复制jar包到WEB_INF,然后右击add

  • 在Tomcat中也需要导入jar包，点击idea右上角Project Structure

部署TomCat界面

  • 选择 Add Cpnfigurtion,，点击加号，选择Tomcat Server服务器

  • 进入Server,点击Configureation选择Tomcat解压路径
    • 进入Open browser选择默认浏览器，并取消勾选After launch，
    • URL显示的为当前服务打开的网址,

  • 在Deployment（部署）中选择Artiface,设定
  • 设定工程路径，在此处更改，URL也会跟随更改

前端访问后端资源

web.xml中注册后端资源路径

  1. 在src中创建一个类，并使其继承HttpServlet类，HttpServlet类是Servlet接口的具体实现类

  2. 重写父类的Service方法，客户端在访问服务器资源时一定会执行此方法。

    • service方法为请求分发处理，在底层，根据浏览器发出的请求作出不同的处理

      @Override
   protected void service(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
     
       //此语句如果前端请求后端资源成功，此语句会打印在控制台
       System.out.println("前端请求后端资源成功");
     
     
   }

  1. 在web文件夹下web.xml中注册后端资源，这是必须要做的

    • 注册后端资源

      
<!--    起始标签-->
    <servlet>

        <!--后端资源的标识  -->
        <servlet-name>hello</servlet-name>


        <!--后端具体资源的权限定类名，简称为全类名  可以在需要注册的类上右击选择copy,copy reference 快捷键 -->
        <servlet-class>com.wufan.servlet.HelloServlet</servlet-class>
    </servlet>
<!--/结束标签    -->

    • 提供资源访问路径

      <!--提供访问资源路径  给前端提供访问后端资源的路径-->
    <servlet-mapping>

        <!--跟上面的标识保持一致-->
        <servlet-name>hello</servlet-name>

        <!--注意添加/，/不存在则找不到资源路径-->
        <!--前端访问的资源路径-->
        <url-pattern> /hello </url-pattern>
    </servlet-mapping>

类中注解注册后端资源路径

  • 可以通过加注解的形式注册后端资源路径（效果等同于web.xmlservlet）

上下文文本对象(域对象)

  • ServletContext是Servlet上下文文本对象，即域对象

  • 域对象：可以像map集合一样存储数据，以键值对形式存储

  • ServletContext对象在整个工程下是单例的，取值范围是整个Web工程，无论在工程内的哪个类都可以写入或者访问创建的域对象

  • Request也是一个域对象，但是这个域对象取值范围只能是一次请求范围之内（只有在转发的时候才能取到数据）

作用范围

ServletContext对象取值作用范围是整个web工程，在不同的类中可以调用此对象获取值或写值

对象获取

//上下文文本对象创建
//方法1 调用getServeltConfig.getServletContext()方法
ServletContext servletContext = getServletConfig.getServletContext();

//方法2 直接调用getServletContext()方法，在底层此方法指向方法1
ServletContext servletContext1 = getServletContext();

读写方法

//写入
//创建完对象后，使用setAttribute(key,value)方法写入,类比map集合的put方法
servletContext.setAttribute("key1","key2");


//读取
//通过键获取值
Object key1 = servletContext.getAttribute("key1");

//如果想要存放数据库数据到域对象，可以创建一个集合，将数据存放集合，再将集合传给域对象

Get.Post请求

Get请求

  • get请求没有请求体，因为把请求封装在地址栏上

Post请求

  • 请求体 ：发送给服务器的数据

响应的HTTP协议格式

  • 响应行
    • 响应的协议和版本号
    • 响应状态码
    • 响应状态描述符
  • 响应头 key:value 不同的请求头，不同的含义
  • 响应体： 回传给客户端的数据

常用请求头说明

  • accept 表示客户端可以接受的数据类型
  • Accept-Language 表示客户端可以接收的语言类型
  • User-Agent 表示客户端浏览器的信息
  • Host 表示请求时的服务器Ip和端口号

常用响应码说明

  • 200 请求成功

  • 302 表示请求重定向

  • 404 表示请求服务器已经收到了，但是数据不存在（请求地址错误）

  • 500 服务器已经收到请求，但是服务器内部错误（代码错误）

HelloServletRequest

  • 每次有请求进入Tomcat服务器，Tomcat服务器就会把请求过来的HTTP协议信息解析好封装到Request对象中。然后传递到service方法(doGet 和 doPost)中给我们使用

    String requestURL = Req.RequestURL();

    String remoHost = req.getRemostHost();

String flag = req.getParameter("list");

if (flag !=null && falg = "list"){

HttpServletRespnose

  • 每次有请求时，Tomcat服务器都会床架一个Response对象传递给Servlet程序使用，HttpServletRequesdt表示请求过来的信息，HttpServletResponse表示所有响应的信息。
  • 设置返回给客户端的信息

往客户端回传数据

  • 字节流 getOutputStream()

    • ·用于下载（传递二进制数据）

  • 字符流 getWriter

    • 用于回传字符串，将数据或者域对象回传给浏览器

      resp.getWriter.println(要传递的数据);

处理响应中文乱码

resp.setContentType("text/html;charset=UTF-8")

转发的两种方式

请求重定向

  • 客户端给服务器发送请求，服务器告诉客户端去新地址访问，跳转到新地址

    //重定向语句，（）内是绝对地址
response.sendRedirect("http://localhost:8080/hello");

请求转发

  • 服务器收到请求后，从一个资源跳转到另一个资源的操作叫请求转发

  • Request具有请求转发方法 request.getRequestDispatcher(/hello1)

    RequestDispatcher r1 = request.getRequestDispatchar("相对路径").forword(requset,response);

  • forword方法用于将请求转发到RequesDspatcher对象封装的资源

区别

  • 重定向是指向绝对路径，一般用于当前网页已经不再使用，可以无限次转发
  • 请求转发指向相对路径，一次转发
