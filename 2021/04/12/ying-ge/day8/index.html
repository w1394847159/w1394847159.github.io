摘要

本文介绍了Mysql数据库，并记录了一些mysql的使用案例，介绍了一对一，一对多，多对一的实现

数据库

概念

  • DataBase 简称DB,通常是一个或一组文件，保存了一些符合特定规格的数据。

表

  • 表示一种结构化文件，可以用来存储特定类型的数据.表有特定的名称，不能重复。
    • 表有几个概念：
      • 列：字段Column
      • 行：表中的记录
      • 主键：，每张表必须要有自己的主键
      • 每一个字段都有：字段名称/字段数据类型/字段约束/字段长度

主键

  • 主键不允许为空，主键必须唯一，在创建主键时，需要为其指定主键，数据类型为整型，不允许为空值
  • 一般多张表之间相互关联，员工表中有部门表的连接，一般称之为外键，而外键一般为另一张表主键。

SQL分类

  • 数据查询语言(DQL-Data Query Language)

    • 代表关键字:select

  • 数据操纵语言(DML-Data Manipulation Language)

    • 代表关键字:insert,delete,update

  • 数据定义语言(DDL-Data Definition Language)

    • 代表关键字:create ,drop,alter,

  • 事务控制语言(TCL-Transactional Control Language)

    • 代表关键字:commit ,rollback;

  • 数据控制语言(DCL-Data Control Language)

    • 代表关键字:grant,revoke.

DQL数据查询语言 select(查询)

  • select用于查询表中的数据
    • select * from 表名; *代表所有数据

DML数据操作语言 insert(添加) delete(删除) update(修改)

  • insert 添加一条数据

    insert tb_students (stu_name,sex,age)values ('张三','1','25');

    第一个 ()内是添加表的每个字段 ，第二个()内添加相应字段的数据

  • update 修改一条数据

    update tb_students set stu_name = '赵六new',sex = 2,age = 35 where stu_name = '赵六'

  • delete 删除一条数据 以主键id删除

    delete from tb_students where id = 2;

order by 排序

  • 排序采用order by子句，order by后面跟排序字段
  • 排序字段可以放多个，多个采用,间隔，如果第一个字段重复，根据第二个字段排序。
  • order by默认采用升序 ，降序则在排序语句后加desc
  • 如果存在where子句,order by必须放到where语句后面
#按照薪水从小到大排序 从小到大在排序语句后加 asc
select * from emp order by sal asc; asc可写可不写

#取得job为MANAGER的员工，按照薪水由小到大排序 *注意where子句在order by前
select *from emp where job = 'manager' order by sal asc;

# 如果有字段存在多个元素，则首先需要按照此字段排序，再按照原有字段排序
#首先按照job排序，再按照sal排序
select *from emp order by job,sal;

#按照薪水从大到小排序 *注：降序需要在排序语句后加 desc;
select *from emp order by sal desc;

//查询员工姓名中包含"L"的员工，并按薪水从大到小排序
select * from emp where ename like '%L%' order by sal desc;

#按照字段位置排序 order by 字段所处位置  *注：要想使用此方法，必须将所有信息全部输出才可排序
#按照薪水排序
select *from emp order by 6; #薪水sal字段处于第六个
#按照姓名排序 姓名字段处于第二个
select *from emp order by 2;

group by 分组查询

  • 分组函数自动忽略空值

  • 排序关键字order by要放在分组关键字group by后面

  • 分组查询主要涉及到两个子句，分别是：group by和having

SQL语句中若有group by 语句，那么在select语句后面只能跟分组函数+参与分组的字段,即只能有一个分组字段。

#取得每个工作岗位的工资合计，要求显示岗位名称和工资合计
select job '岗位', sum(sal) '岗位获得所有薪水' from emp group by job;

#取得每个工作岗位的平均工资，要求显示岗位名称和平均工资，然后按照薪资升序 *注：字段如果使用了别名，在其余地方都需要使用别名
select job '岗位', avg(sal) sal1  from emp group by job order by sal1 asc;

#按照工作岗位和部门编码分组，取得的工资合计 *注：group by后只能有一个分组字段
select job,deptno,sum(sal) from emp group by job,deptno;

//按照工作岗位和部门编码分组，取得的工资合计 
select job ,deptno,sum(sal) from emp group by job,deptno;

分组查询的执行顺序

  • 根据条件查询数据
    • 分组group by
      • 采用having过滤，取得正确的数据

having 再过滤

  • having 关键字 对分组后的数据再次进行过滤

    //取得每个方位的平均工资大于2000
select job,avg(sal) avg from emp group by job having avg > 2000;

分页 limit

  • 分页的关键字一定是出现在整个sql语句的后面（即在order by后面）
  • 表达： limit startIndex,length
    • startIndex 起始位置，下表从0开始
    • length 取几条数目
select *from emp limit 起始索引位置,分页单位
#起始索引从0开始，分页单位即从索引位置开始往后几条。

外连接

  • 外连接需要将while ---> on
  • 外连接是多表查询的一种，把其中一张表看成主表，另外一张看成从表
  • 外连接在查询过程中，要求主表的数据是必须全部出现的，如果匹配字段为null，这个null会直接显示，不会被过滤掉。(主表类比java的key,必须要全部有值，从表类比java的value，不一定赋值)
  • sql99语法在from后面where只能出现一次
左外连接

  • 主表 left (outer) join 从表

  • 如果A表连接B表，把一张表看做主表，一张表看做副表

  • 主表的内容全部显示，副表的内容可以不用全部显示，如果副表没有字段匹配，则返回null

    #左连接
SELECT a.ENAME '员工姓名',b.ENAME '领导姓名'
FROM emp a left join emp b
on a.MGR = b.EMPNO
右外连接
  • 从表 right join 主表
    • 右连接以右边的表为准和左边的表比较，和右表相等的不相等的都会显示出来，左表符合条件的显示，不符合条件的不显示。
SELECT d.DNAME , e.DEPTNO
FROM emp e RIGHT JOIN dept d
ON e.DEPTNO = d.DEPTNO
WHERE e.DEPTNO IS NULL #此时e表作为副表，如果没有字段与主表匹配，则显示为空，通过判空查找结果

select语句总结

一个完整的select语句格式：

select 字段
from 表名
where …….
group by ……..
having …….(就是为了过滤分组后的数据而存在的—不可以单独的出现)
order by ……..

执行顺序：

  1. 首先执行where语句过滤原始数据
  2. 执行group by进行分组
  3. 执行having对分组数据进行操作
  4. 执行select选出数据
  5. 执行order by排序

原则：能在where中过滤的数据，尽量在where中过滤，效率较高。having的过滤是专门对分组之后的数据进行过滤的。

	Name varchar(20) not null comment'姓名',
	Sex varchar(4) comment'性别',
    Birth year comment'出生年份',
    Department varchar(20) comment'院系',
    Address varchar(50) comment'家庭住址',
      primary key (Id)
      );
      
CREATE TABLE score
      ( Id int(10) not null auto_increment unique comment'编号',
	Stu_id int(10) not null comment'学号',
	C_name varchar(20) comment'课程名',
    Grade int(10) comment'分数',
      primary key (Id)
      );
