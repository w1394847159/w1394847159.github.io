摘要

Spring是一个轻量级框架，主要用于与前端交互，IOC控制反转让对象与对象之间有效解耦。

Spring框架

Spring简介

  • 面向表现层，主要与前端进行交互

  • Spring是容器，可以帮忙管理很多可重用的组件

  • 最大的目的就是使JAVA EE开发更加容易

  • Spring 方便解耦，简化开发

  • Spring容器中的实例默认是单例的，是可重用的组件，内存地址为一份

Spring 核心

  • IOC容器
    • IoC负责创建与管理系统对象，在此基础上扩展功能
  • AOP面向切面编程

正转

  • (传统开发方式)

  • 正转 ： 在软件开发中，聚合在类中的实例，是由开发人员创建的，对象实例创建的控制权在开发人员手上

    • 即在程序中new对象
    • 坏处： 使类与类之间产生了依赖关系，使得程序的执行效 率变低
    private TestDao dao = new TestDaoimpl();

  • 即聚合实例的引用时，使用构造方法或set方法，在类调用处传入对象

IOC控制反转

  • IOC控制反转，Inversion of Control

  • 由代理人创建与管理对象，消费者通过代理人来获取对象

    代理人 对象 消费者

    中介 房东 租房客

  • 在IOC容器中，持久层(B)通过xml文件或注解的形式注入到业务层(service)

目的

  • 利用Ioc容器，让对象与对象之间有效解耦
    • 解释：使用配置文件进行对象创建及配置，不需要修改代码，也不需要new对象
  • 加入IOC容器将对象同一管理，让对象关联变为弱耦合
    • 消费者获取对象都是从IOC容器中获取

特点

  • 将程序中写死的文本剥离出来，放入配置文件中

    <!--在Ioc容器启东时，自动由spring实例化Apple对象，取名sweepApple放入到容器中，即全局标识-->
    <bean id="sweetApple" class="com.wufan.spring.ioc.entity.Apple">
        <property name="title" value="红富士"></property>
        <property name="origin" value="欧洲"></property>
        <property name="color" value="红色"></property>
    </bean>

DI依赖注入

  • DI Dependency Injection具体技术实现，在运行时 让程序创建对象

  • Java中通过反射实现对象注入，即本质为反射技术

  • 在类中直接创建对象的方式改成给该引用指定构造方法，程序里不存在new关键字,这也被称为依赖注入，

    程序由主动地创建对象变成被动的接收对象，这是控制反转的一种思想，对象创建的控制权交给了客户端

    //聚合接口的引用
private TestDao dao;

    /*在类中直接创建对象的方式改成给该引用指定构造方法，
    程序里不存在new关键字,这也被称为依赖注入，
    程序由主动地创建对象变成被动的接收对象，这是控制反转的一种思想
    对象创建的控制权交给了客户端*/
    public TestServiceimpl(TestDao dao) {
        this.dao = dao;
    }

声明式事务

  • 在spring中需要对数据库数据进行提交或者回滚操作
  • 声明式事务规则，进入方法打开事务，成功时提交，运行异常时回滚
  • @Transactional是声明式事务的注解，一般情况下都使用注解方式进行操作

  • 使用声明时事务只需要在需要的方法上加上**@Transactional注解**

//@Transactional声明时事务，加在类上所有方法都会开启事务，加在方法上单个方法开启事务
//Propagation.REQUIRED 对于需要事务的方法自动开启，
//rollbackFor = RuntimeException.class 当遇到什么异常时回滚数据，一般情况下实际上是遇到运行时异常RuntimeException时回滚数据
@Transactional(propagation = Propagation.REQUIRED,rollbackFor = RuntimeException.class)
public class UserDao {


    //@Transactional()
    public void getUser(){

        System.out.println("我叫用户");
    }

全局标识符

  • 使用bean创建对象时存在 name和id

    <bean id="apple6" name="apple7"/>

共同点

  • bean id 与 name 都是设置对象在IoC容器中唯一标识
  • 两者在同一个配置文件中都不允许重复
  • 两者允许在多个配置文件中出现重复，新的对象会覆盖原有的重名对象

不同点

  • id 要求更严格，一次只能定义一个对象标识(推荐)
  • name更宽松，一次允许定义多个对象标识

  • 如果对象是单例，那核心容器在加载的时候，对象就会创建，核心容器关闭的时候（调用close方法），对象就会销毁

注解配置Bean

注解种类

用于创建对象

创建对象注解对象当于：<bean id="" class="">

  • @Component :（一般用于实体类中）
    • 配置管理的资源
    • 把资源让 spring 来管理。相当于在 xml 中配置一个 bean。
  • ****@Controller：一般用于表现层的注解。
  • @Service：一般用于业务层的注解。
  • ****@Repository：一般用于持久层的注解。
用于注入数据

相当于

<property name="" ref=""> <property name="" value="">

  • @Autowired（根据类型注入）

    • 自动按照类型注入。当使用注解注入属性时，set 方法可以省略。

      它只能注入其他 bean 类型。

      当有多个类型匹配时，使用要注入的对象变量名称作为 bean 的 id，在 spring 容器查找，找到了也可以注入成功。找不到就报错。

  • @Qualifier（不能单独使用） 根据名称注入

    • 如果匹配多个类型一致的对象，将无法选择具体注入哪一个对象
      使用@Qualifier()注解告诉spring容器自动装配哪个名称的对象。
     //根据类型注入
    @Autowired 
//按名称注入，不能单独使用
    @Qualifier("accountServiceImpl")  //配合Autowired使用
    private AccountService accountService;

  • @Resource（根据id注入的） 最常使用的自动装配注解

    • 直接按照 Bean 的 id 注入。它也只能注入其他 bean 类型。
    • @Resource(“name”) ** @Autowired + @Qualifier(“name”)

  • @Value（用于注入基本数据类型和String类型）

  • @Scope

    • 指定 bean 的作用范围。

  • @PostConstruct用于指定初始化的方法

  • @PreDestroy用于指定销毁的方法

  • @Transactional 声明式事务管理 编程中使用的注解

    • 只被应用到 public 方法上

核心注解

配置创建

@Configuration  //该注解确认当前类是一个配置类，用于替代applicationContext.xml
public class Config {

    @Bean  //Java Config利用方法创建对象，将方法返回对象放入容器，beanId - 方法名
            //等同于 <bean id="userDao" ...> 用于创建实例
    public UserDao userDao(){

        UserDao userDao = new UserDao();

        return userDao;
    }

    /**
     *
     * @param userDao  在JavaConfig中，依赖注入是通过参数形式注入，
     *                 Java首先会生成bean，当检测到参数名与beanId相同时，自动将对应的bean赋值给参数
     *                 实现依赖注入，等同于注解@Autowired
     * @return
     */
    @Bean  //依赖注入时，先按name尝试注入，name不存在按类型注入
    public UserService userService(UserDao userDao){

        UserService userService = new UserService();

        userService.setUserDao(userDao);

        return userService;
    }

    @Bean
    public UserController userController(UserService userService){

        UserController userController = new UserController();

        userController.setUserService(userService);

        return userController;
    }
}
  • 依赖注入时，先按name尝试注入，name不存在按类型注入

模拟底层解耦

模拟SPring Ioc基于注解底层解耦过程：

  • 在xml中开启包扫描，Spring会去扫描指定包下面的组件
  • 通过反射机制去过滤组件上是否有spring提供的注解(Controller,Service..)
  • 如果有注解，Spring会通过反射机制创建实例，保存到容器中,然后Spring会根据反射机制检查该类中是否有属性需要注入（也就是属性上是否有Spring提供的注解）
  • 如果有注解，Spring会根据反射机制进行注入

AOP

  • 参照案例

    1. D:\Data\JAVA\MianShi_Practise\Spring_AOP

概述

  • AOP:Aspect Oriented Programming 面向切面编程

  • 程序执行前，程序执行后都可以执行一些扩展功能，这种扩展功能称为切面

  • 连接点 JoinPoint 通过连接点可以获取目标类/方法的信息，即真正需要执行的方法称为目标方法
    • 在aop切面类中的切面方法中使用
  • PointCut 切点，使用execution表达式描述切面的作用范围
    • 在spring-bean配置文件中配置AOP时使用

作用

  • 程序运行期间，在不修改源码的情况下对程序行为进行扩展

优势

  • 减少重复代码
  • 提高开发效率
  • 维护方便

2. 创建切面类和切面方法

  1. 创建切面类，包含对原有代码的扩展方法

    package com.wufan.aspect;

import org.aspectj.lang.JoinPoint;

import java.text.SimpleDateFormat;
import java.util.Date;

/**
 * 方法的切面  即扩展
 */
//切面类
public class MethodAspect {

    //切面方法，用于扩展额外功能
    //JoinPoint 连接点  通过连接点可以获取目标类/方法的信息，即真正需要执行的方法称为目标方法
    public void printExecutionTime(JoinPoint joinPoint){

        // SSS代表毫秒级别
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss SSS");

        String now = sdf.format(new Date());

        //getTarget 查找目标执行的类   getSignature 查找目标执行的方法
        String className = joinPoint.getTarget().getClass().getName();//获取目标类的名称
        String methodName = joinPoint.getSignature().getName();

        System.out.println("------>" + now + ":" + className + "." + methodName);

    }
}

3.配置切面类 Aspect Bean

  • 在spring-bean配置文件中通过 aop:config标签进行aop配置

  • 注意导入aop的schema (官网中在Appendix中的The aop Schema)

    <!--配置切面类-->
    <bean id="methodAspect" class="com.wufan.aspect.MethodAspect"></bean>

4.定义PointCut

  • PointCut定义与切面类配置都在同一个spring-bean配置文件中

    <aop:config> <!--用于配置aop-->
  
     <!--Pointcut 切点，使用excution表达式描述切面的作用范围-->
     <!--execution(public * com.wufan..*.*(..)) 说明切面作用在com.wufan包下的所有类的所有方法上-->
     <aop:pointcut id="pointcut" expression="execution(public * com.wufan..*.*(..))"/>
     <!--定义切面类 aop:aspect用于定义切面类-->
     <aop:aspect ref="methodAspect">
         <!--before通知，前置通知，代表目标方法运行前先执行methodAspect.printExecutionTime()方法-->
         <aop:before method="printExecutionTime" pointcut-ref="pointcut"></aop:before>
  
     </aop:aspect>
 </aop:config>

切点表达式

execution(public * com.wufan..*.*(..)) 等介于

​ public void com.wufan.......

  • * 为 通配符 –> 用于对应一切可能出现的情况
  • .. 为 包通配符 –> 只要隶属于com.wufan，无论是一层还是多层，都可以进行查找
  • (..) 为 参数通配符 –> 去匹配任意形式，不限个数，不限形式的方法

5.配置Advice 通知

  • 见上部分代码，与pointCut共同写在aop:config中

通知类型

  • 通知即为在什么时间去执行切面的方法

前置通知

Before Advice

  • 在目标方法执行前执行
    • 可用作对目标方法权限的控制

返回后

After Returning Advice

  • 在目标方法返回数据后执行的方法

异常通知

After Throwing Advice

  • 在目标方法抛出异常后执行
    • 类似于try catch关键字

后置通知

After Advice

  • 在目标方法运行后执行

最强大通知！！环绕通知

Around Advice

  • 自定义通知的执行时机，可以决定目标方法是否运行

单元测试junit

  • 在spring中，可以集成单元测试模块，单元测试莫夸可以去测试任何模块，而不需要写主函数
  • 单元测试在底层封装了main函数(运行器)

使用注解

  • @Test 为模块添加单元测试注解，在方法上添加

  • @RunWith(SpringJUnit4ClassRunner.class) 替换运行器 添加在类上

    • 括号内 ： SpringJUnit4ClassRunner.class Spring提供的运行器，可以在程序运行时，动态获取核心容器对象

  • ContextConfiguration() 告诉运行器配置文件时是基于什么配置

优点

  • 使用单元测试junit，无需创建核心容器对象，可直接对方法进行操作
  • 需要调用的属性使用Autowired注明

样例

@RunWith(SpringJUnit4ClassRunner.class)  //替换运行器为Spring运行器，在程序运行时，可以动态获取核心容器对象
@ContextConfiguration(locations = "classpath:beans.xml")  //指定配置文件  必须与RunWith搭配使用
public class Test02 {

    //由于已经使用RunWith动态创建核心容器对象，可以直接提取核心容器对象进行使用
    //不需要使用new关键字进行对象创建
    @Autowired  //从容器中获取代理对象
    AccountService accountService;

    //使用junit单元测试运行加钱扣钱方法
    @Test
    public void save(){

        accountService.transfer();
        accountService.saveAccount();

    }
}

术语

术语 名称 作用
Joinpoint 连接点 连接点指被拦截到的点，拦截实际上是增强的意思。
实际上是目标类中的方法，代理类可以访问的方法都可以称为连接点
Pointcut 切入点 目标类中需要被增强的方法
Advice 通知/增强 拦截到Joinpoint后要做的事情就是通知。前置通知，后置通知，异常通知，最终通知，环绕通知
Target 目标对象 代理的目标对象
Weaving 织入 指把增强应用到目标对象来创建新的代理对象的，spring采用动态代理织入
Proxy 代理 一个类被AOP织入增强后，就产生一个结果代理类
Asperct 切面 切入点（目标类中需要被增强的方法）和通知（怎么增强）之间的关系

Joinpoint(连接点)

  • 拦截实际上是增强的意思

连接点指被拦截到的点。在spirng中，这些点指的是方法，实际上是目标类中的方法，代理类可以访问的方法都可以称为连接点

Pointcut(切入点)

  • 指对那些连接点在代理中进行增强，被增强的连接点称为切入点，没有被增强的依旧称为连接点

Advice

  • 通知/增强 即怎么增强，整个增强的过程称为通知
  • 在调用目标方法之前调用（织入）的内容为前置通知
  • 在调用目标方法之后调用的内容为后置通知
  • 在catch语句块中的内容称为异常通知
  • 在finally语句块中的内容称为最终通知
  • 整个invoke方法执行的过程称为环绕通知

Target

  • 目标对象

Weaving

  • 织入
  • 增强应用实际上就是执行增强目标方法的代码

Proxy

  • 代理
  • 一个类被AOP织入增强后产生一个结果代理类

Aspect

  • 切面
  • 切入点和通知之间的关系
  • Spring中配置AOP实际上是配置切入点和通知的关系
