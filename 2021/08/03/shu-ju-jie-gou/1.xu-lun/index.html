1.绪论

发表于 2021-08-03 更新于 2021-08-18 分类于 数据结构

摘要

本文是数据结构的基础知识，简要介绍数据结构的基本概念，记录数据的物理结构与逻辑结构以及它们的区别。

绪论

  • 数据结构是 相互之间存在一种或多种特定关系的数据元素集合

  • 数据结构是一门研究非数值计算的程序设计问题中的操作对象，以及它们之间关系和操作等相关问题的学科

程序设计= 数据结构 + 算法

基本概念

数据

数据： 描述客观事务的符号，是计算机中可以操作的对象，可以被计算机识别，并输入给计算机处理的符号集合。

  • 数据包括：
    • 整型、实型等数值类型
    • 字符、声音、图像、视频等非数值类型

数据元素

  • 数据元素： 组成数据的、有一定意义的基本单位，计算机中通常作为整体处理。也被称为记录。

    在人类中 数据元素是 人

    畜类中 数据元素是 牛，马，羊等

数据项

  • 数据项： 一个数据元素可以由若干个数据项组成

    数据元素人 有眼，耳，鼻等数据项，也可以有姓名、年龄、性别等数据项

    具体有哪些数据项，视做的系统决定！！

  • 数据项是不可分割的最小单位

    • 但实际生活中，一般以 人 这种数据元素为最小单位，而不会以其年龄作为最小单位

数据对象

  • 数据对象：是性质相同的数据元素的集合，是数据的子集
    • 性质相同，即数据元素具有相同数量和类型的数据项，例如人都有姓名、生日等相同的数据项

数据结构

  • 结构 ： 简单理解就是关系，不同数据元素之间不是独立的而是存在特定关系，这些关系称为结构
  • 数据结构： 是相互之间存在一种或多种特定关系的数据元素的集合

结构

逻辑结构

  • 逻辑结构： 数据对象中数据元素之间的相互关系

  • 逻辑结构针对具体问题，

    是为了解决某个问题，

    在对问题理解的基础上，

    选择合适的数据结构表示数据元素之间的逻辑关系。

  • 逻辑结构有四种

    1. 集合结构
    2. 线性结构
    3. 树形结构
    4. 图形结构

注意事项

注意事项： 使用示意图标识数据逻辑结构时

  • 将每一个数据元素看做一个节点，用圆圈表示
  • 元素之间逻辑厝用节点之间的连线表示，这个关系有方向，用带箭头的连线表示。

集合结构

  • 集合结构： 集合结构中的数据元素除了同属一个集合，它们之间没有其它关系，各元素之间是平等的
    • 共同属性：同属于 一个集合

线性结构

  • 线性结构： 线性结构中的数据元素是 一对一的关系

    • 数组、链表、队列、哈希表（用到了数组）

树形结构

  • 树形结构： 树形结构中数据元素之间存在 一对多 的层次关系

    • 二叉树、AVL树、红黑树、B树、堆、Trie（字典树）、哈夫曼树、并查集

图像结构

  • 图形结构： 图形结构的数据元素是 多对多的关系

物理结构

  • 物理结构： 指数据的逻辑结构在计算机中的存储形式。
  • 数据元素存储结构：
    • 顺序存储
    • 链式存储

顺序存储结构

  • 顺序存储结构： 把数据元素存放在地址连续的存储单元里，数据间的逻辑关系和物理关系一致。

    • 实际上就是 排队占位。 大家按顺序排好，每人占一小段空间
    • 数组为 顺序存储结构

链式存储结构

  • 链式存储结构： 把数据元素存放在任意存储单元，这组存储单元可以连续或不连续。

  • 链式存储结构只需要关心下一个数据的位置就可以了

    • 链式存储很灵活，数据存在哪里不重要，有一个指针存放了相应地址就能找到它

区别

  • 逻辑结构 面向问题
  • 物理结构 面向计算机
    • 基本的目标：将数据及其逻辑关系存储到计算机的内存中

数据类型

数据类型

  • 数据类型： 一组性质相同的值的集合以及定义此集合上的一些操作的总称

分类

数据类型在C语言中可以分为两类：

  • 原子类型： 不可以再分解的基本类型，包括整型、实型、字符型
  • 结构类型： 由若干个类型组合而成，可以再分解。例如整型数组由若干个整型数据组成。

抽象数据类型

  • 抽象： 抽取出事务具有的普遍的性质
    • 例如测量小车的速度，则将小车抽象为一个点，而不需要考虑小车长什么样
  • 抽象数据类型(Abstract Data Type ADT)： 一个数学模型以及定义在该模型上的一组操作

抽象数据类型标准格式

  • 描述抽象数据类型的标准格式：
ADT 抽象数据类型名

Data 
		数据元素之间逻辑关系的定义

Operation 
		操作1
				初始条件
				操作结果描述
		操作2
				...
		操作n
				...
endADT
