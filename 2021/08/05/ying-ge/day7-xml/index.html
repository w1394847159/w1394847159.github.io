day7_xml

发表于 2021-08-05 分类于 后端 ， java

摘要

本文介绍XML的特性，作用，语法，还介绍了Cookie以及Session会话等

XML

  • XML 是Extensible Markup Language简称，中文翻译为可扩展标记语言。

XML特性

  • 平台无关性
  • 语言无关性
  • 系统无关性

XML作用

  • 用来保存数据，而且这些数据具有自我描述性
  • 可以作为项目或者框架或者模块的配置文件
  • 作为网络传输数据的格式
  • 作为编程语言与编程语言之间数据交互的桥梁

XML语法

  • 有且只有一个根元素

  • xml文档声明必须放在头部（文档第一行）

    • xml声明

      <?xml version="1.0" endoding="utf-8" ?>

  • 所有标签必须成对出现

  • XML标签严格区分大小写

  • XML必须正确嵌套

  • XML中的属性值必须加引号

    <student id = "110102">

  • XML中，一些特殊符号需要使用“实体”

  • XML中可以使用适当的注释

XML元素

  • XML元素指的是开始标签到结束标签的部分
  • 一个元素包含:
    • 其他元素
    • 文本
    • 属性
    • 以上的混合

XML命名规则

  • 元素的名字可以包含字母，数字和其他字符
  • 元素的名字不能以(xmlm,XML)等开头
  • 元素的名字不能以数字或者标点符号开头
  • 元素的名字不能包含空格

XML文档节点类型

  • document ：文档，代表整个文档（DOM树的根节点）

  • element ：元素，表示一个元素

  • attribute： 属性，代表一个属性

  • PCDATA (Parsed Character Data): 文本

  • comment ： 注释，代表一个注释

  • DOCTYPE ：主要研制文档内容的正确性

  • ENTITIEs 实体

XML解析技术

  • dom解析技术是W3C组织制定的，所有编程语言都对这个解析技术使用自己语言的特点进行实现
  • java中一般使用 dom4j
  • jdom在dom基础上进行封装，dom4j在jdom基础上进行封装
  • 在以后学习的各大框架，都是通过dom4j进行解析

XML解析过程

  1. 根据流对象读取xml文件
  2. 获取需要解析数据的根标签
  3. 获取子标签，并且根据子标签获取数据
JAVA中

//创建流对象
SAXReader saxReader = new SAXReader();

//读取xml文件，创建Document对象
//Docunment xml的上下文文本对象
Document document = saxReader.read("src/com/wufan/xml/students.xml");

//获取根标签对象
Element rootElement = document.getRootElement();

//获取子标签 element获取的是一个子标签，elements获取的是所有子标签，返回类型为集合,  （）内为子标签名

List<Element> student = rootElement.elements("student")

//遍历所有子标签
for(Element element : student){
    
    String name = element.elementText("name")
}

Cookie(面试重点)

  • Cookie 是服务器通知客户端保存键值对的一种技术。
  • 有了cookie，可以把数据保存在浏览器
  • 每个Cookie大小不能超过4kb
  • cookie以键值对的形式存储数据
  • cookie 的 key如果相同，在浏览器端会覆盖

Cookie创建

  • 创建cookie对象

    req.getCookies()

Cookie执行原理

  • 创建一个cookie,通知浏览器保存，实际上是通过http协议响应头的方式来通知浏览器的
    • 浏览器如果在响应头中发现cookie,则将cookie放入浏览器

Cookie 值修改

  1. 创建一个相同的Key用以覆盖

    //创建cookie
        Cookie cookie = new Cookie("key1","newValue1");

        //通知客户端保存
        response.addCookie(cookie);

  2. 方案2

    1. 先查找到需要修改的Cookie对象
    2. 调用setValue()方法赋予新的Cookie值
    3. 调用response.addCookie()通知客户端保存修改
    //调用工具类(工具类是自己创建的)
        Cookie cookie1 = FindCookieUtils1.find(cookies, "key4");

//调用setValue方法赋予新的Cookie值      
cookie1.setValue("newValue4");

        //通知浏览器保存
        response.addCookie(cookie1);

Cookie生命控制

  • 控制方法：setMaxAge()

  • 正数，表示在指定的秒数后过期

  • 负数，表示浏览器一关，Cookie 就会被删除（默认值是-1） 即Session级别的

  • 零，表示马上删除 Cookie

EI表达式

在jsp中可以直接调用cookie的值

${cookie.键名.值}

Session会话

  1. Session就一个接口(HttpSession)

  2. Session就是会话，用来维护客户端和服务器之间关系的技术

  3. 每个客户端都会有自己的一个Session会话

  4. Session会话中，经常用来保存用户登录之后的信息

  5. Session有生命周期，在默认的生命周期范围内，无论创建多少个Session,永远指向同一个地址

创建Session会话

  • 创建Session会话

    //创建Session会话
HttpSession session = req.getSession();

  • 判断是否是刚创建的Session

    boolean aNew = session.isNew();

  • 得到Session的会话id值

    • Session会话id 实际上就是Cookies的value值
    String id = session.getId();

获取Session域数据

  • 存数据

    req.getSession().setAttribute("key","balue");

  • 取数据

    • 取数据时根据key取value值，也可以在EI表达式中使用${key}
    req.getSession().getAttribute("key")

Sesson生命周期

  • Sesson对象具有生命周期，默认生命周期是30min

修改生命周期

  • 通过配置web.xml修改超时时间

     <session-config>

<!--timeout内的时间实际上是以分钟计算 40*60-->
        <session-timeout>40</session-timeout>
    </session-config>

  • 通过在代码中修改

    //创建一一个Sessiond对象
        HttpSession session = req.getSession();

        //代码内修改session超时时间  配置内修改在web.xml中

//此处（）内是以秒计算
session.setMaxInactiveInterval(60*60);

//获取超时时间
int maxInactiveInterval = session.getMaxInactiveInterval();
让当前session立即失效
session.invalidate();
