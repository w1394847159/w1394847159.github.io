二叉树

发表于 2021-08-17 更新于 2021-08-18 分类于 数据结构

本文包含了二叉树的概念、性质以及二叉搜索树

二叉树

概念

  • 二叉树是只有两个结点的树形，由于左右的不同，可以分为左子树和右子树。一个结点最多有2个结点，A为一个结点

特点

  • 每个结点最多只能有两个子树，二叉树中不存在度大于2的结点

  • 左子树和右子树有顺序，次序不能颠倒。

  • 即使树中某一个结点只有一棵子树，也要区分是左子树还是右子树。如下图，左边为左子树，右边为右子树。

  • 二叉树的五种基本形态
    1. 空二叉树
    2. 只有一个根结点
    3. 根结点只有左子树
    4. 根结点只有右子树
    5. 根结点既有左子树又有右子树

特殊二叉树

1. 斜树

  • 所有结点只有左子树的二叉树叫左斜树，所有结点只有右子树的二叉树叫右斜树，两者统称为斜树。

    • 线性表结果也可以理解为树的一种特殊表现形式

2. 满二叉树

  • 在一棵二叉树中，如果所有分支结点都存在左子树和右子树，并且所有叶子都在同一层，这样的二叉树称为满二叉树

  • 满二叉树特点
    1. 叶子（最后结点）只能出现在最下面一层。
    2. 非叶子结点的度一定是2，即有两条树枝
    3. 同样深度的二叉树中，满二叉树的结点个数最多，叶子数最多。

3. 完全二叉树

对一棵具有n个结点的二叉树按照层序编号，编号为i(1 < i < n) 的结点与同样深度的满二叉树中编号为i的结点在二叉树中位置完全相同，并且编号连续，则称为完全二叉树。下图上边为完全二叉树，下边不是完全二叉树。

​

  • 完全二叉树特点
    1. 叶子结点只能出现在最下两层
    2. 最下层叶子一定集中在左部连续位置（满足编号连续）
    3. 倒数二层，如果有叶子结点，一定都在右部连续位置
    4. 结点度为1，则该结点只有左孩子，不存在只有右子树的情况
    5. 同样结点数的二叉树，完全二叉树深度最小

二叉树性质

  1. 在二叉树的第i层上最多有$2^{i-1}$​个结点​

  1. 深度为k的二叉树至多有$2^k$-1个结点

    深度为k即有k层二叉树

  2. 任何一棵二叉树T，如果终端结点数为$n_0$​，度为2的结点数为$n_2$，则$n_0 = n_2+1$

    ​ 终端结点数为叶子结点数，即没有子结点的结点

    • 如上图： n0结点数共有H,I,J,F,G 5个度为0的叶子结点，E为度为1的结点，A,B,C,D4个度为2的结点。

      总和为5+4+1 = 10.

  3. 具有n个结点的完全二叉树的深度为$[log_2n]+1$​ ([x]表示不大于x的最大整数)。

  • 第一条，i = 1时，为根结点，没有双亲；i >1时，例如结点5，其双亲为[5/2] = 2,结点8，双亲为[8/2] = 4.
  • 第二条，例如结点4，2x4 = 8，小于结点总数10，所以有左孩子，为8;同样，结点6，2x6 = 12，大于结点数10，没有左孩子。
  • 第三条，例如结点5， 2x5+1 = 11，大于结点数10，没有右孩子；同样，结点3，2x3+1 = 7，小于结点数10，有右孩子，右孩子为2x3+1 = 7.

二叉树存储结构

顺序存储结构

  • 顺序存储结构只能用于完全二叉树，对于其他的二叉树会造成浪费。

  • 一般二叉树没有向完全二叉树一样依照顺序编号，则容易空出很多的内存，但是顺序存储时空白位置也会占据一定空间，造成对存储空间的浪费。

二叉链表

  • 二叉链表，即是将指向结点的左孩子和右孩子的指针与结点存放在一起的链表，如下图，data是数据域，lchild和rchild都是指针域。

二叉搜索树

  • 二叉搜索树也叫二叉查询树，二叉排序树，以当前节点为根，左右子树特点为：左大右小 如下图，左边的都是比50小的，右边都是比50大的

1. 查找

二叉树根据左大右小的特点，查找一个元素时，从根节点出发：

  1. 如果查找的元素比当前节点小，到左子树找
  2. 如果查找的元素比当前节点大，到右子树找
  3. 如果查找的元素等于当前节点，则找到了
  4. 如果到叶子节点也没找到，则该元素不存在在二叉树中

2. 插入

插入元素也有要求：

  1. 插入的树为空树，即无节点，直接就创建根节点

  2. 插入的树非空树：

    1. 插入的元素比当前节点小，则到左子树插入，找到左子树为null处，插入元素
    2. 插入的元素比当前节点大，则到右子树插入，找到右子树为null处，插入元素
    3. 插入的元素和当前节点相等，说明元素已经存在了，直接返回

3. 删除

3.1 找到前继、后继节点

  • 删除之前需要找到删除点的前继节点，后继节点。
  • 前继节点表示比当前节点小的最大值
  • 后继节点是比当前节点大的最小值

  • 寻找前继节点，当前节点左转一次，右转到底

  • 寻找后继节点，当前节点右转一次，左转到底

3.2 根据子节点数量删除

删除之前要找到删除的节点，找不到则不删除

找到之后，分三种情况：

  1. 没有子节点，即当前节点为叶子节点，直接删除

  2. 只有一个子节点，使用子节点连接到删除节点的父节点

  3. 有两个子节点，使用前或后继节点作为替换的节点，对删除节点进行数据的替换，原来的后继节点删除
